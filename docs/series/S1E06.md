# S1E06 — Typst Download Route

**Why this matters (hiring manager lens):**  
Design managers can now grab a clean, versionable representation of the resume straight from the web tier, unblocking asset reviews without screenshots.

## What we built
- `apps/web/templates/resume.typ` plus a typed renderer that converts `ResumeDocument` data into Typst markup.
- `buildTypstSource` + `renderTypstPdf` coverage to keep the exports aligned with the on-screen layout.
- `/api/resumes/[slug]/typst` and `/api/resumes/[slug]/pdf` download routes that stream the `.typ` source or a compiled PDF straight from Next.js.
- Inline download buttons on `/r/[slug]` so reviewers can grab either format without memorizing URLs.

## Key decisions
- Anchor the first iteration inside Next.js to avoid juggling CLI dependencies across services; reevaluate if we later need PDF binaries or caching.

## How to run it
```bash
make up
npm run dev --prefix apps/web
open http://localhost:3000/api/resumes/main/typst
```

## Demo script (3–5 min)
1) Start the dev server and load `/r/main` to prove the resume still renders as HTML.
2) Click the new in-page buttons to fetch Typst and PDF downloads; show that the PDF opens cleanly.
3) Open the `.typ` file in an editor (or compile locally) to show the exported sections match the UI.

## What’s next
- Cache compiled PDFs/KV artifacts so repeated downloads don’t hit WASM every time, then explore piping those assets to storage for sharing.
