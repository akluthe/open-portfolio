# S1E07 — Resume Tailoring (Master + Overlay)

**Why this matters (hiring manager lens):**  
One canonical work history, many application-specific cuts — edit the master once and every tailored version reflects it, so a referral-ready resume is a few checkboxes away instead of a copy-paste fork that drifts.

## What we built
- A `profiles` table and a dumb-storage `.NET` API (`/profiles` list, `GET/PUT/DELETE /profiles/{slug}`) that stores small JSON overlays referencing a master `baseSlug`.
- `resolveTailoredResume(master, profile)` — a pure, defensive resolver in `@resume-platform/shared-types` that applies headline/summary overrides, hides/keeps experience entries and individual bullets, hides/reorders skills, and drops whole sections, always returning a schema-valid `ResumeDocument`.
- A public tailored view at `/t/[slug]` plus `/api/profiles/[slug]/{pdf,typst}` — reusing the exact same `ResumeView`, `buildTypstSource`, and `renderTypstPdf` pipeline as master resumes.
- Admin tailoring UI: list/create at `/admin/tailoring` and a master-aware editor at `/admin/tailoring/[slug]` with include/exclude toggles for entries, bullets, skills, and sections.

## Key decisions
- Resolve overlays in the TypeScript tier, not C# — one resolver feeds the HTML view, the Typst route, and the PDF route; the API stays a dumb JSON store.
- Index-based addressing (experience by array index, highlights by index) to avoid forcing stable IDs into existing master docs; the resolver ignores out-of-range indices. Tradeoff: reordering the master may require re-checking a profile's selections.
- Authoritative validation is the Zod `tailoringProfileSchema` before PUT; the API does only a light `name` + `baseSlug` structural check.

## How to run it
```bash
make up
make dev
open http://localhost:3000/t/databank-engmgr
```

## Demo script (3–5 min)
1) Open `/admin/tailoring`, create a tailoring (name + slug + baseSlug), and land in its editor.
2) Toggle off an experience entry and a couple of bullets, override the headline, save, and watch `/t/{slug}` and `/api/profiles/{slug}/pdf` reflect the cut.
3) Edit the master at `/admin/{baseSlug}`, then reload `/t/{slug}` to prove the tailoring is an overlay, not a copy.
